### Q.10 Why is it important to connect a local repository with a remote one before pushing code?

### Q.11 What is the purpose of the `git pull` command, and how does it differ from `git fetch`?

**Answer:**

* **`git pull`** downloads the latest changes from the remote repository and automatically merges them into the local repository.
* **`git fetch`** only downloads the latest changes but does **not** merge them. You can review the changes before merging.

**Difference:**

| `git pull`                                  | `git fetch`                         |
| ------------------------------------------- | ----------------------------------- |
| Downloads and merges changes automatically. | Downloads changes only.             |
| Updates the local branch immediately.       | Requires manual merge later.        |
| Easier for regular updates.                 | Better for reviewing changes first. |
